# Промпт для Claude-агента: извлечение событий

Этот промпт даётся каждому параллельному субагенту в шаге 3 пайплайна. Используется модель `haiku` (Claude Haiku 4.5) — задача структурированная, дешевле и быстрее.

## Использование

После `python scripts/3_extract_events.py 6 15` есть 6 файлов `batch_results/agent_<i>_input.txt` со списками video_id. Запустите 6 параллельных агентов, каждому скормите промпт ниже с подставленным `<N>`.

В Claude Code это выглядит так (одно сообщение с 6 блоками `Agent`):

```python
Agent(
    description="Extract events round X (haiku 1)",
    subagent_type="general-purpose",
    model="haiku",
    prompt="<тело промпта с N=1>"
)
# … ещё 5 одинаковых вызовов с N=2..6
```

## Промпт (подставить N — номер агента)

```
Обрабатываешь транскрипты криминального шоу «Дорожный патруль» (Москва, 1996-2008). Каждый выпуск содержит 3-7 событий с конкретными адресами.

ВХОД: `<абсолютный путь>/batch_results/agent_<N>_input.txt` (по одному video_id в строке)
ТРАНСКРИПТЫ: `<абсолютный путь>/transcripts/<video_id>.txt`
ВЫХОД: `<абсолютный путь>/batch_results/agent_<N>_output.json`

Структура транскрипта: первые 3 строки — `# <Title>`, `# Date: ...`, `# URL: ...`. Далее `[HH:MM:SS] текст`.

ИЗВЛЕКАЙ для каждого события с адресом:
- timecode: HH:MM:SS — момент начала рассказа об этом событии
- address: точный адрес (улица+дом+Москва, или район+Москва, или «N-й км Х шоссе, Москва»)
- event_type: один из {ДТП, криминал, пожар, теракт, несчастный случай, поджог, коммунальная авария, авиакатастрофа, другое}
- description: 1 короткое предложение с именами/возрастом/деталями если упомянуты

ПРОПУСКАЙ:
- Рекламу спонсоров (Инком, Пантера, Феникс, Европрестиж, фильмы)
- Розыск пропавших, призывы звонить по телефонам
- Статистику за неделю
- В выпусках «Сводка за неделю» — события, очевидно дублирующие выпуски той же недели
- События вне Москвы и Подмосковья
- Если адреса нет в транскрипте — пропусти событие (НЕ выдумывай)

ВАЖНО: НЕ занижай число событий. Если в выпуске 5–15 происшествий с адресами, извлеки все.

ФОРМАТ JSON:
{
  "processed_video_ids": ["id1","id2",...все ID из input.txt],
  "events": [
    {
      "video_id": "...",
      "video_title": "из строки # Title",
      "video_date": "DD.MM.YYYY (из заголовка в скобках)",
      "video_url": "https://www.youtube.com/watch?v=...",
      "timecode": "HH:MM:SS",
      "address": "...",
      "event_type": "...",
      "description": "..."
    },
    ...
  ]
}

ОБЯЗАТЕЛЬНО: processed_video_ids включает ВСЕ video_id из input.txt (даже если событий не нашлось — спецвыпуски, реклама).

Когда закончишь — отчитайся: видео обработано / событий извлечено.
```

## Заметки по качеству

- **Haiku 4.5** хорошо справляется с этой задачей и в ~10× дешевле Sonnet 4.6. Расход ~100K токенов на агента (15 видео).
- **Дедупликация** на уровне `(video_id, timecode)` — `_apply_extracted_events.py` пропускает повторы.
- **«Сводка за неделю»** (выпуски в воскресенье) дублируют события дневных выпусков. Агенты получают инструкцию пропускать явные дубли, но не идеально — финальная дедупликация по адресу и описанию делается отдельно (TODO).
- Если агент возвращает мало событий — увеличить `max_tokens` или дать большие батчи на ввод (но не больше 20 видео — иначе теряется качество).
