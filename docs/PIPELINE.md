# Полный пайплайн извлечения

Этот документ описывает что делает каждый шаг и почему так. Для просто-запустить-всё см. README.

## Архитектура

```
YouTube playlist (638 видео)
    │
    ▼  scripts/1_download_subs.py  (yt-dlp + cookies + Node.js EJS solver)
[ subs/ — 599 .ru.vtt файлов ]
    │
    ▼  scripts/2_parse_vtt.py  (умная склейка с удалением перекрытий)
[ transcripts/ — 599 .txt файлов с [HH:MM:SS] разметкой ]
    │
    ▼  scripts/3_extract_events.py  (готовит batch_results/agent_<i>_input.txt)
    │  + 6 параллельных Claude haiku субагентов с промптом из docs/AGENT_PROMPT.md
    │  → batch_results/agent_<i>_output.json
    │
    ▼  scripts/_apply_extracted_events.py
[ data/events_dataset.csv — 3432 строк ]
    │
    ▼  scripts/4_geocode.py  (Nominatim, 1 req/sec, кэш в data/geocode_cache.json)
    ▼  scripts/4b_geocode_retry.py  (умные варианты для пропущенных)
[ data/events_dataset.csv с lat/lon ]
    │
    ▼  scripts/5_export_geojson.py
[ data/events_dataset.geojson — 2729 features ]
```

## Шаг 1: Скачивание субтитров

**Проблема №1:** Видео в плейлисте 18+ → нужны cookies залогиненного аккаунта.

**Проблема №2:** YouTube требует решать JS-челлендж (anti-bot). yt-dlp использует Node.js + загружаемый с GitHub скрипт `ejs:github` ([yt-dlp/ejs](https://github.com/yt-dlp/ejs)).

**Проблема №3:** Cookies истекают примерно каждые 50 запросов. Скрипт детектит истечение по строке `Sign in to confirm your age` в выводе yt-dlp и **ждёт обновления** `cookies.txt` (по mtime файла) — без перезапуска.

**Резумная стратегия:** на каждом батче проверяем что .vtt-файл существует и `>100 байт`. Уже скачанное пропускаем.

## Шаг 2: Парсинг VTT

YouTube auto-subs выглядят как «скользящее окно» — каждая фраза появляется в нескольких подряд идущих VTT-блоках со сдвигом. Если просто склеить — получится текст с тройными повторами.

**Алгоритм `remove_overlap`:** перебираем длины суффикса предыдущей фразы от max до 1, ищем самый длинный, который равен префиксу новой. Удаляем перекрытие. Это нормально работает, но иногда теряет короткие повторы (3-4 буквы).

Время-разбивка: каждые ~10 секунд начинаем новую строку. Это даёт читаемый текст с таймкодами достаточной точности для агентов.

## Шаг 3: Извлечение событий

Главный challenge — масштаб (599 транскриптов × ~25 КБ).

**Что не работало:**
- Ручное чтение каждого транскрипта — занимает 70+ ходов разговора, упирается в контекст
- Один большой запрос с всеми транскриптами — невозможно (>1 МБ контекста)

**Что работает:** **параллельные субагенты**. Каждый Claude Haiku обрабатывает 15 видео независимо в своём контексте. 6 агентов параллельно × 15 видео = 90 видео за раунд.

Промпт-инжиниринг важен:
- **Чёткие правила пропуска** (реклама, розыск, дубли в недельных сводках) — иначе агент извлекает шум
- **Запрет придумывать адреса** — иначе галлюцинирует «улица Иванова» где её нет
- **Требование вернуть ВСЕ video_id** в `processed_video_ids` — даже если событий не нашлось (рекламные блоки, спецвыпуски)
- **Явная просьба не занижать число** — без этого Haiku возвращает 1-2 события на 15-минутный выпуск

Агент пишет JSON в `batch_results/agent_<N>_output.json`. Главный поток подхватывает все файлы.

## Шаг 4: Геокодинг

Nominatim (OpenStreetMap) — бесплатный, но 1 запрос/сек по [usage policy](https://operations.osmfoundation.org/policies/nominatim/).

**Стратегия:**
1. **Кэш:** `geocode_cache.json` keyed по адресу как написано в датасете. Уникальных адресов ~3000 (из 3432 событий). Resumable — перезапуск продолжает.
2. **Нормализация:** срезаем «квартира», «корпус», «этаж» — Nominatim спотыкается на детализации.
3. **Fallback в `4b_geocode_retry.py`:**
   - Срезаем POI-определения (`гостиница X`, `ТЦ Y`, `школа Z`)
   - Заменяем `дом N` → просто `, N`
   - `ул.` → `улица`
   - Финальный fallback: только название улицы + «Москва»

Достигнутое покрытие: **80% адресов**, **79% событий**.

**Что не геокодируется:** старые/закрытые/переименованные объекты («ДЭЗ Чертаново», «магазин Орбита»), искажения распознавания.

**Альтернатива:** Yandex Geocoder API — лучше работает с российскими адресами, но платный (~$0.1 за 1000 запросов).

## Шаг 5: Экспорт GeoJSON

Простая трансформация CSV → GeoJSON FeatureCollection. Свойства фичи = все колонки CSV кроме `lat`/`lon`. Включаются только события с координатами.

Готовый файл импортируется в:
- Leaflet / MapLibre — построить веб-карту
- QGIS — для GIS-анализа
- Mapbox — для production-карт
- Kepler.gl — для аналитики
